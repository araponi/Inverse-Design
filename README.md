## Overview

This repository is a **paper companion** for the mirror-model strategy developed for **data-driven inverse design of magnesium hydroxide precipitation**. The workflow couples a mechanistic **CFD–PBM** formulation with a **deep-learning mirror model** to infer effective kinetic parameters from experimentally accessible characteristic particle sizes.

The key point is that the neural network **does not replace the CFD–PBM solver itself**. Instead, it replaces the **iterative optimization loop** that would otherwise require repeated forward CFD–PBM evaluations during parameter identification. In this way, the mirror model acts as a fast approximation of the **inverse mapping** between particle descriptors and kinetic parameters.

## Scientific Context

Magnesium hydroxide precipitation is strongly affected by:
- feeding strategy,
- supersaturation dynamics,
- hydrodynamics,
- nucleation, growth, and agglomeration kinetics,
- morphology-dependent shape effects.

In the reference work, the mechanistic model is used to describe semi-batch precipitation under different single-feed and double-feed operating strategies. A deep-learning-aided inverse-design step is then used to identify the kinetic parameter set consistent with the experimentally observed particle-size descriptors.

## What the Mirror Model Does

The mirror model is trained on a synthetic database generated through repeated CFD–PBM simulations. Once trained, it receives as input:

- operating conditions,
- characteristic particle sizes: `d10`, `d21`, `d32`, `d43`,

and returns the corresponding **effective kinetic parameters** controlling:

- primary nucleation,
- crystal growth,
- agglomeration.

This strategy avoids repeated optimization-driven calls to the forward solver and makes inverse parameter identification substantially faster.

## Workflow

1. **Generate synthetic data** from the forward CFD–PBM model.
2. **Filter and preprocess** the data before training.
3. **Train the mirror model** on operating conditions and characteristic particle sizes.
4. **Interrogate the trained network** using experimental particle-size descriptors.
5. **Validate the identified parameter set** on independent operating conditions.

## Neural-Network Setup

The mirror model described in the reference implementation uses a fully connected feed-forward neural network with:

- **6 hidden layers**
- neurons per layer: **176, 240, 64, 64, 64, 64**
- hidden activation: **ReLU**
- output activation: **linear**
- optimizer: **Adam**
- learning rate: **1e-3**
- loss function: **mean absolute error**
- batch size: **60**
- maximum epochs: **2000**

### Preprocessing

Before training:

- physically inconsistent samples are removed by enforcing  
  `d10 < d21 < d32 < d43`
- selected input particle-size descriptors are **log-transformed**
- selected kinetic outputs spanning multiple orders of magnitude are **log-transformed**
- both inputs and outputs are scaled to **[0, 1]** using MinMax normalization

### Data Split

The full dataset is divided into:
- **90% training**
- **10% hold-out / validation**

The best model is selected through checkpointing based on the minimum validation loss.

## Inputs and Outputs

### Inputs to the mirror model
- feed concentrations
- flow rates
- initial conditions
- feeding time
- characteristic particle sizes (`d10`, `d21`, `d32`, `d43`)

### Outputs from the mirror model
The network returns the six effective kinetic parameters used in the CFD–PBM model:

- `A`
- `B`
- `k_g`
- `g`
- `C1`
- `A_p`

These correspond to the minimum kinetic structure required to describe:
- nucleation,
- growth,
- agglomeration.

## What This Repository Is For

This repository is intended to support:

- scientific transparency of the mirror-model workflow,
- reproducibility of the inverse-design procedure,
- future extensions to additional precipitation systems and reactor configurations,
- paper-companion dissemination of the modelling and machine-learning strategy.

## Important Modelling Notes

- The mirror model is trained **only on synthetic data** generated by the forward CFD–PBM framework.
- Therefore, the neural network does **not add new physics**.
- The inferred quantities should be interpreted as **effective kinetic parameters** within the adopted model structure.
- Particle morphology is **not predicted from first principles**. A morphology-dependent shape factor is assigned separately once the particle habit is experimentally known.
